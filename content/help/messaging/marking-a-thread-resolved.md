**What this is for:** Closing off a conversation once the patient's question has been answered, so the shared inbox stays tidy for the whole team.

## When to resolve a thread

Resolve a thread when the conversation is genuinely finished — the patient's question is answered and no reply is expected. Resolving tells the rest of the team that no one needs to pick this thread up.

## How to mark a thread resolved

1. Open the patient's thread in the shared inbox.
2. Check the conversation is complete and you have sent any final reply.
3. Click **Resolve** in the thread header.
4. The header now shows a **Resolved** badge instead of the button.

## What resolving changes

A resolved thread:

- Drops out of the default inbox view and the unread count.
- Still appears under the **Resolved** filter.
- Can still receive replies — staff can keep messaging on it.

If the patient sends a new message, the thread becomes active again in the list. Resolving is audit-logged with your name and the time.

:::tip
If you are not sure a conversation is over, leave it open. An unresolved thread is easy for a colleague to spot; a prematurely resolved one is easy to miss.
:::

:::patient-sees
Nothing changes for the patient — resolving is staff-internal. The patient can still message the clinic on the same thread at any time.
:::
